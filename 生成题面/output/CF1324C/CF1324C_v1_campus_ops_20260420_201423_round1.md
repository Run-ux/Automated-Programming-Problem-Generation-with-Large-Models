# 校园推车调度

> Variant: `CampusCartNav_Ext` / `v1` / theme `campus_ops`

## 题目描述

校园后勤部需要调度一辆智能推车，将物资从仓库（位置 0）运送到终点广场（位置 n+1）。走廊被划分为 n 个连续的检查点，编号为 1 到 n。每个检查点 i 都设有一个单向指示牌，记录在字符串 s 中（s[i] 为 'L' 或 'R'）。

推车在任意位置 i (0 ≤ i ≤ n) 准备出发时，必须遵守以下规则：
1. 若 i = 0 或 s[i] == 'R'，推车只能向右跳跃至位置 j (j > i)。
2. 若 s[i] == 'L'，推车只能向左跳跃至位置 j (j < i)。
3. 无论方向如何，单次跳跃的跨度 |j - i| 不得超过给定的最大步长 D。

推车一旦落在某个位置，就会根据该位置的指示牌决定下一次跳跃方向。你的任务是判断推车能否成功抵达 n+1。若能，请给出一条合法的落脚点序列；若不能，请找出一个“阻塞区间” [l, r] 作为不可达的局部证据。该区间需满足：区间内所有检查点的指示牌均为 'L'，且区间长度 r - l + 1 ≥ D。这表示任何跨度不超过 D 的跳跃都无法从左至右跨越该段。

## 输入格式

第一行包含一个整数 t，表示测试用例的数量。
每个测试用例包含两行：
第一行包含两个整数 n 和 D，分别表示检查点数量和最大跳跃步长。
第二行包含一个长度为 n 的字符串 s，仅由字符 'L' 和 'R' 组成，表示 1 到 n 号检查点的指示方向。

## 输出格式

对于每个测试用例，输出一行结果：
若存在合法路径，按顺序输出从 0 到 n+1 的落脚点编号，相邻数字间用空格分隔。
若无法抵达，输出两个整数 l 和 r，表示阻塞区间的左右端点（1 ≤ l ≤ r ≤ n）。

## 数据范围与限制

- 1 ≤ t ≤ 10^4
- 1 ≤ n ≤ 2×10^5
- 1 ≤ D ≤ n+1
- 所有测试用例的 n 之和不超过 2×10^5
- 时间限制：1.0 秒
- 空间限制：256 MB

## 样例

### 样例 1

**输入**

```text
2
5 3
RLLRL
5 2
RLLLR
```

**输出**

```text
0 1 4 6
2 4
```

**说明**

第一个用例中，最长连续 'L' 段为位置 2~3，长度为 2，小于 D=3，因此可达。一条合法路径为 0→1→4→6。
第二个用例中，位置 2~4 均为 'L'，连续长度为 3，大于等于 D=2。任何从左侧出发的跳跃跨度均无法越过该区间，故输出阻塞区间 2 4。

### 样例 2

**输入**

```text
1
3 2
RRL
```

**输出**

```text
0 2 4
```

**说明**

最长连续 'L' 段仅位置 3，长度为 1，小于 D=2。推车可从 0 跳至 2（跨度 2），再从 2 跳至 4（跨度 2），成功抵达终点。

## 说明

1. 路径序列必须严格递增，且首项为 0，末项为 n+1。
2. 阻塞区间 [l, r] 必须对应原字符串中一段连续的 'L'，且满足 r - l + 1 ≥ D。若存在多个满足条件的区间，输出任意一个即可。
3. 所有位置编号均基于 0 到 n+1 的整数索引，字符串 s 的索引对应 1 到 n。
