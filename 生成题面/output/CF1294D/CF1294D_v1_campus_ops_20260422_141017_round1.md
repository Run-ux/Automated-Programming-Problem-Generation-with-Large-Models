# 校园社团资源调度

> Variant: `CAMPUS_CANONICAL_MEX` / `v1` / theme `campus_ops`

## 题目描述

校园社团联合会正在规划新学期的活动资源。初始时资源池为空。接下来会有 q 次社团入驻申请，每次申请会提交一个初始资源需求值 v_i。由于后勤仓库的物资调配机制，每个社团最终分配到的资源编号 v_i' 必须满足：v_i' >= 0，且 v_i' ≡ v_i (mod x)（x 为固定的调配步长）。
为了公平与高效，联合会规定了一套规范分配原则：在所有满足上述同余与非负约束的分配方案中，必须选择能使当前已分配资源集合的 MEX（最小未出现非负整数）最大的方案；若存在多个方案均能使 MEX 最大，则必须选择字典序最小的分配序列 (v_1', v_2', ..., v_k')。该原则要求：对于任意两个原始位置 i < j，若其分配值不同，则必须优先将较小的可用资源编号分配给 i。规范顺序的优先级高于同余约束的自由度。
你的任务是模拟这一过程：每次新社团入驻后，立即确定当前所有社团的规范分配值，并输出结果。

## 输入格式

第一行包含两个正整数 q 和 x，分别表示入驻申请次数与调配步长。
接下来 q 行，每行包含一个非负整数 v_i，表示第 i 次申请提交的初始资源需求值。

## 输出格式

共输出 q 行。第 i 行包含 i 个非负整数，表示前 i 次申请对应的规范分配值 v_1', v_2', ..., v_i'，相邻整数之间用单个空格分隔。

## 数据范围与限制

- 1 <= q <= 400000
- 1 <= x <= 400000
- 0 <= v_i <= 10^9
- 时间限制：2.0 秒
- 空间限制：512 MB
- 保证所有测试点的输出总字符数不超过 5×10^6，建议使用快速输入输出

## 样例

### 样例 1

**输入**

```text
3 2
1
0
2
```

**输出**

```text
1
1 0
1 0 2
```

**说明**

第一次申请需求为 1，模 2 余 1。当前 MEX 为 0，但余 0 类无可用位置，故 MEX 保持为 0。为满足字典序最小，位置 1 分配其同余类中最小合法值 1。第二次申请需求为 0，模 2 余 0。余 0 类有位置 2 可用，将其分配给 0，MEX 推进至 1。此时余 1 类无新位置，MEX 停止。位置 1 保持 1，位置 2 为 0。第三次申请需求为 2，模 2 余 0。余 1 类仍无位置，MEX 保持 1。位置 3 分配最小合法值 2。最终序列字典序最小且 MEX 最大。

### 样例 2

**输入**

```text
4 3
0
3
1
4
```

**输出**

```text
0
0 3
0 3 1
0 3 1 4
```

**说明**

第一次申请需求 0 模 3 余 0，位置 1 分配 0，MEX 推进至 1。第二次申请需求 3 模 3 余 0，余 1 类无位置，MEX 保持 1，位置 2 分配最小合法值 3。第三次申请需求 1 模 3 余 1，余 1 类有位置 3 可用，分配 1，MEX 推进至 2。第四次申请需求 4 模 3 余 1，余 2 类无位置，MEX 保持 2，位置 4 分配最小合法值 4。输出序列严格遵循字典序最小与 MEX 最大化原则。

## 说明

本题要求输出精确的规范分配数组，无需取模。字典序比较按从左到右顺序进行。分配值必须严格满足同余与非负条件。若某位置未被用于填充 MEX 前缀，则应分配其同余类中大于等于当前 MEX 的最小非负整数，以保证全局字典序最小。输出数组本身即为可校验的解，其结构隐含了全局约束的满足性。
