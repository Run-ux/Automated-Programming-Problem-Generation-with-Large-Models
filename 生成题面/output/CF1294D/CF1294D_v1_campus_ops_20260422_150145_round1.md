# 稳健的社团排期

> Variant: `CF1294D_robust_mex` / `v1` / theme `campus_ops`

## 题目描述

某高校后勤中心负责统筹全校社团的活动场地分配。本学期共有 $q$ 批社团依次提交入驻申请，每批申请携带一个初始资源编号 $v$。为便于统一调度，中心规定所有社团在分配场地时，必须遵循“模 $x$ 同余编队”原则：即社团最终使用的场地编号 $v'$ 必须满足 $v' \equiv v \pmod x$，且场地编号必须为非负整数。

然而，场地调度过程中难免出现临时变动（如设备维护、突发占用等）。中心经过历史数据评估，给出一个扰动阈值 $\delta$：对于任意一个模 $x$ 的余数类 $r$，实际能够投入使用的场地数量，最多会比当前已收到的该余数类申请总数少 $\delta$ 个。这意味着，若当前已收到余数为 $r$ 的申请共 $\text{cnt}[r]$ 个，则最坏情况下的实际可用名额为 $\text{usable}[r] = \max(0, \text{cnt}[r] - \delta)$。

中心希望制定一个稳健的排期计划。对于当前已经收到的所有申请，请计算一个最大的整数 $K$，使得**无论扰动如何发生**（只要满足上述损失上限），中心都能通过合法的场地分配，让编号为 $0, 1, 2, \dots, K-1$ 的场地全部被成功占用。换言之，对于任意余数 $r \in [0, x-1]$，区间 $[0, K-1]$ 中模 $x$ 余 $r$ 的整数个数 $\text{slots}(K, r)$ 均不得超过该类最坏情况下的可用名额 $\text{usable}[r]$。

系统会严格按申请到达的顺序依次处理。你需要对每一批新到达的申请，实时基于累计收到的所有资源，计算并输出当前满足条件的最大 $K$。

## 输入格式

第一行包含三个整数 $q, x, \delta$，分别表示申请批次数、同余编队步长和扰动阈值。
接下来 $q$ 行，每行包含一个整数 $v_i$，表示第 $i$ 批社团的初始资源编号。输入按时间顺序给出，需依次处理。

## 输出格式

输出共 $q$ 行，第 $i$ 行包含一个整数，表示处理完前 $i$ 批申请后，当前最大的稳健保底编号上限 $K$。

## 数据范围与限制

- $1 \le q, x \le 400000$
- $0 \le \delta \le 10^9$
- $0 \le v_i \le 10^9$
- 时间限制：2.0 秒
- 空间限制：512 MB

## 样例

### 样例 1

**输入**

```text
4 2 0
0
2
1
3
```

**输出**

```text
1
1
3
4
```

**说明**

第1批收到0，cnt=[1,0]，可用[1,0]。区间[0,0]中余0需1个，满足，K=1。
第2批收到2(≡0)，cnt=[2,0]，可用[2,0]。若K=2，区间[0,1]中余1需1个，但可用为0，不满足，故K保持1。
第3批收到1，cnt=[2,1]，可用[2,1]。K=3时，区间[0,2]中余0需2个，余1需1个，均满足，K=3。
第4批收到3(≡1)，cnt=[2,2]，可用[2,2]。K=4时，余0需2个，余1需2个，均满足，K=4。

### 样例 2

**输入**

```text
5 3 1
0
3
1
4
2
```

**输出**

```text
0
1
1
2
2
```

**说明**

δ=1，每个余数类最坏损失1个名额。
第1批：收到0，cnt=[1,0,0]，可用[0,0,0]。无法满足K≥1，K=0。
第2批：收到3(≡0)，cnt=[2,0,0]，可用[1,0,0]。K=1满足，K=2时余1需1>0不满足，故K=1。
第3批：收到1，cnt=[2,1,0]，可用[1,0,0]。K=2时余1需1>0，故K保持1。
第4批：收到4(≡1)，cnt=[2,2,0]，可用[1,1,0]。K=2满足，K=3时余0需2>1不满足，故K=2。
第5批：收到2，cnt=[2,2,1]，可用[1,1,0]。K=3时余0需2>1，故K保持2。
