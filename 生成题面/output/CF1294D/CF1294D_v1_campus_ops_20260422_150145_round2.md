# 校园时段调度

> Variant: `CAMPUS_SLOT_ASSIGNMENT` / `v1` / theme `campus_ops`

## 题目描述

大学社团联合会正在规划新学期的活动场地。现有 $K$ 个连续时段，编号为 $0, 1, \dots, K-1$。同时，有若干支社团队伍陆续抵达，每支队伍携带一个唯一的资源编号 $a_i$。

为了公平起见，调度规则规定：编号为 $a_i$ 的队伍只能被安排在时段 $v$，且必须满足 $v \equiv a_i \pmod x$（其中 $x$ 为给定的步长参数）。此外，每个时段最多只能容纳一支队伍，且所有队伍必须恰好填满 $0$ 到 $K-1$ 的每一个时段，不允许遗漏或重复。

队伍会按顺序依次抵达。你需要在每支队伍抵达后，立即判断当前已抵达的队伍集合能否在满足上述规则的前提下，完成对 $0 \sim K-1$ 时段的完美覆盖。

若能完成，请输出一种具体的分配方案（即每个时段对应的队伍编号）；若不能，请输出一个“极小冲突余数类集合”，用于证明当前资源池无论如何分配都无法达成目标。该集合需满足：集合内所有余数类对应的时段需求总数严格大于当前可用队伍总数，且从中移除任意一个余数类后，剩余部分的需求均不超过可用队伍数。

## 输入格式

第一行包含三个正整数 $q, x, K$，分别表示查询次数、步长参数和目标时段总数。
接下来 $q$ 行，每行包含一个非负整数 $a_i$，表示第 $i$ 次查询新增的队伍资源编号。

## 输出格式

对于每次查询，输出两行或三行：
第一行输出 `YES` 或 `NO`，表示当前资源池能否完成覆盖。
若为 `YES`，第二行输出 $K$ 个整数，第 $j$ 个整数表示时段 $j$ 分配到的队伍资源编号。
若为 `NO`，第二行输出一个整数 $m$ 表示极小冲突集合的大小，第三行输出 $m$ 个互不相同的整数，表示冲突的余数类（按升序排列）。若存在多个满足条件的极小集合，输出字典序最小的一个即可。

## 数据范围与限制

- 1 ≤ q, x, K ≤ 400000
- 0 ≤ a_i ≤ 10^9
- 时间限制：2.0 秒
- 空间限制：512 MB
- 数据保证所有输出 YES 的查询中 K 的总和不超过 400000

## 样例

### 样例 1

**输入**

```text
3 2 3
2
3
4
```

**输出**

```text
NO
1
0
NO
1
0
YES
2 3 4
```

**说明**

第一次查询后仅有资源2（余数0），时段0,1,2的需求分别为余数0需2个、余数1需1个。余数0供给不足，输出极小冲突集{0}。第二次查询后资源2,3（余数0,1），余数0仍缺1个，冲突集仍为{0}。第三次查询后资源2,3,4（余数0,1,0），各余数供给均满足需求，成功分配。

### 样例 2

**输入**

```text
2 4 5
5
9
```

**输出**

```text
NO
1
0
NO
1
0
```

**说明**

目标时段0~4，步长4。各余数需求为：0需2个(0,4)，1需1个(1)，2需1个(2)，3需1个(3)。第一次查询资源5（余数1），余数0供给为0，不足，输出{0}。第二次查询资源9（余数1），余数0仍为0，不足，输出{0}。

## 说明

1. 极小冲突集定义：设集合 S 为若干余数类的集合，若 S 中所有余数类在 0~K-1 内的时段需求总数严格大于当前可用资源总数，且 S 的任意真子集均不满足该严格大于条件，则称 S 为极小冲突集。由于本题同余约束的独立性，任意一个供给不足的余数类均可单独构成极小冲突集，输出字典序最小的一个即可。
2. 模运算定义：v ≡ a_i (mod x) 表示 v 与 a_i 除以 x 的余数相同，且 v ≥ 0。
3. 输出方案时，第 j 个数字对应时段 j 分配的资源编号，资源编号需与输入完全一致。
