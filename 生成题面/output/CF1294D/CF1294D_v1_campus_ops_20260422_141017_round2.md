# 社团排期计划

> Variant: `CampusOps_CanonicalSchedule` / `v1` / theme `campus_ops`

## 题目描述

校园运营中心正在为各学生社团安排活动场地。现有 $q$ 个社团依次提交申请，第 $i$ 个社团提交的基础期望时间为 $v_i$。由于场地调度规则，每个社团最终被分配的时间 $t_i$ 必须满足 $t_i \equiv v_i \pmod x$ 且 $t_i \ge 0$。此外，为了保证活动不冲突，任意两个社团的分配时间必须互不相同。

运营中心希望所有社团的分配时间集合 $\{t_1, t_2, \dots, t_k\}$ 的 MEX（即集合中未出现的最小非负整数）尽可能大。在 MEX 达到最大的前提下，分配时间序列 $[t_1, t_2, \dots, t_k]$ 必须字典序最小。若某位置未被用于填充 MEX 前缀，则分配其同余类中大于等于当前 MEX 的最小非负整数。

每当有新社团加入时，系统会基于当前所有已申请的社团重新计算满足上述条件的规范分配方案。历史社团的分配时间允许被动态调整，以维持全局最优性。你需要对每次查询（即每加入一个新社团后），输出当前所有社团的规范分配时间序列。

## 输入格式

第一行包含两个正整数 $q$ 和 $x$，分别表示社团申请总数和调度步长。
接下来 $q$ 行，每行包含一个非负整数 $v_i$，表示第 $i$ 个社团的基础期望时间。

## 输出格式

共 $q$ 行。第 $i$ 行输出 $i$ 个非负整数，表示前 $i$ 个社团的规范分配时间 $t_1, t_2, \dots, t_i$，相邻整数间用空格分隔。

## 数据范围与限制

- $1 \le q \le 400000$
- $1 \le x \le 400000$
- $0 \le v_i \le 10^9$
- 时间限制：2.0 秒
- 空间限制：512 MB

## 样例

### 样例 1

**输入**

```text
3 2
0
0
1
```

**输出**

```text
0
0 2
0 2 1
```

**说明**

第1个社团期望0，模2余0，分配最小非负值0。集合{0}，MEX=1，输出0。
第2个社团期望0，模2余0，0已被占，分配同余类下一可用值2。集合{0,2}，MEX=1，字典序最小序列为[0,2]，输出0 2。
第3个社团期望1，模2余1，分配最小非负值1。集合{0,1,2}，MEX=3，字典序最小序列为[0,2,1]，输出0 2 1。

### 样例 2

**输入**

```text
4 3
1
1
1
1
```

**输出**

```text
1
1 4
1 4 7
1 4 7 10
```

**说明**

所有社团期望模3均余1。只能分配1,4,7,10...。由于0始终无法被分配，MEX恒为0。为满足字典序最小，按顺序依次分配同余类中最小可用值。

### 样例 3

**输入**

```text
3 1
5
2
8
```

**输出**

```text
0
0 1
0 1 2
```

**说明**

步长x=1时，模1余数均为0，等价于分配任意非负整数。为最大化MEX并保证字典序最小，系统紧凑填充0,1,2。每次新增社团均填补当前MEX空缺。

## 说明

1. MEX 定义：给定非负整数集合中未出现的最小非负整数。
2. 字典序比较：从左至右逐位比较，首次出现较小数值的一方字典序更小。
3. 分配策略核心：本题要求输出完整的分配向量，仅维护同余类资源数量无法确定具体槽位归属。必须采用精确槽位映射机制，按社团ID升序贪心分配最小合法槽位，并在冲突时快速定位同余类下一空闲位置。
4. 动态重计算：每次查询后，系统会全局重算规范解。历史分配值可能因新社团加入而调整，但算法可通过维护各余数类的连续空闲槽位指针实现均摊高效更新。
