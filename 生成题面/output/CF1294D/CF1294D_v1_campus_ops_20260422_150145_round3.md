# 校园时段分配

> Variant: `CAMPUS_COVER_CERT` / `v1` / theme `campus_ops`

## 题目描述

校园社团活动需要借用教室。现有 x 个循环时段（编号 0 到 x-1）。每支队伍 i 抵达时提交一个基础偏好时间 a_i。由于排课弹性，该队伍可以被安排到任意时段 s，只要满足 s ≡ a_i (mod x) 或 s ≡ a_i + 1 (mod x)。教务系统会不断收到新队伍的申请，并间歇性发起查询：给定一个目标区间长度 K，问能否从当前已抵达的队伍中选出恰好 K 支，为时段 0, 1, ..., K-1 各分配一支不同的队伍（即完美覆盖）。若能，输出具体方案；若不能，根据 Hall 定理，必然存在某个子区间 [L, R]（0 ≤ L ≤ R < K），其能覆盖的队伍总数严格小于区间长度 R - L + 1。此时需输出字典序最小的冲突区间 [L, R]。

## 输入格式

第一行包含两个正整数 x 和 q，分别表示时段周期长度与操作总数。
接下来 q 行，每行描述一次操作：
- 1 a_i：表示一支新队伍抵达，其基础偏好时间为 a_i。队伍编号按抵达顺序从 1 开始依次递增。
- 2 K：表示一次覆盖查询，要求判断能否覆盖时段 0 至 K-1。

## 输出格式

对于每次 2 K 查询，输出一组结果：
- 若存在合法分配方案，第一行输出 YES。随后 K 行，每行输出两个整数 id s，表示将编号为 id 的队伍分配至时段 s。任意合法方案均可。
- 若不存在合法方案，第一行输出 NO。第二行输出两个整数 L R，表示字典序最小的冲突区间（即优先最小化 L，其次最小化 R）。

## 数据范围与限制

- 1 ≤ x, q ≤ 4×10^5
- 1 ≤ a_i, K ≤ 10^9
- 时间限制：2.0 秒
- 空间限制：512 MB
- 保证所有输出 YES 的查询中 K 的总和不超过 4×10^5

## 样例

### 样例 1

**输入**

```text
3 4
1 0
1 1
2 2
2 3
```

**输出**

```text
YES
1 1
2 0
NO
0 2
```

**说明**

队伍1偏好0，可覆盖时段0或1。队伍2偏好1，可覆盖时段1或2。查询K=2需覆盖[0,1]，分配1→1、2→0即可。查询K=3需覆盖[0,2]，仅2支队伍，任意长度为3的区间供给均不足，字典序最小冲突区间为[0,2]。

### 样例 2

**输入**

```text
4 5
1 0
1 2
1 3
2 3
2 4
```

**输出**

```text
YES
3 0
1 1
2 2
NO
0 3
```

**说明**

前三支队伍分别偏好0、2、3。查询K=3时，队伍3→0、1→1、2→2可完美覆盖。查询K=4时，需覆盖[0,3]，但仅有3支队伍，全局供给不足，最小冲突区间为[0,3]。

## 说明

1. 模运算定义：s ≡ a (mod x) 表示 s 除以 x 的余数等于 a 除以 x 的余数。时段编号为非负整数。
2. 字典序定义：区间 [L1, R1] 字典序小于 [L2, R2] 当且仅当 L1 < L2 或 L1 = L2 且 R1 < R2。
3. 冲突区间性质：在本题的相邻余数耦合约束下，若无法完成覆盖，必然存在至少一个连续子区间 [L, R] 满足供给不足。输出要求保证该区间是字典序最小的极小违反集。
4. 方案输出：当输出 YES 时，任意满足约束的分配方案均被接受，无需保证字典序或特定顺序。
