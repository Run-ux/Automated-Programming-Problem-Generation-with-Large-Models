# 校园通道权限码

> Variant: `CAMPUS_NET_WITNESS` / `v1` / theme `campus_ops`

## 题目描述

某高校正在升级校园内部的安全管网。管网由 N 个设施节点和 N-1 条双向通道构成，整体呈树形结构。在规划图纸上，每条通道已按录入顺序被依次编号为 1 到 N-1。为了统一门禁标准，任意两个终端设施（即仅连接一条通道的叶子节点）之间的通行权限码（通道边权）按位异或和必须严格为 0。
后勤部门希望尽可能简化权限码的种类以降低维护成本。在满足上述异或约束且使用最少种类权限码的前提下，为了符合审计规范，要求按通道编号 1 到 N-1 顺序排列的权限码序列字典序严格最小。请为每条通道分配合适的正整数权限码，并输出该规范序列。

## 输入格式

第一行包含一个整数 N，表示设施节点数量。
接下来 N-1 行，每行包含两个整数 u 和 v，表示一条通道连接的两个设施节点。通道按输入顺序自动获得编号 1 至 N-1。

## 输出格式

输出一行 N-1 个正整数，表示编号 1 到 N-1 的通道所分配的权限码，相邻数字间用空格分隔。

## 数据范围与限制

- 3 <= N <= 100000
- 1 <= u, v <= N，保证输入构成一棵合法的树
- 权限码必须为正整数
- 时间限制：2.0 秒
- 空间限制：512 MB

## 样例

### 样例 1

**输入**

```text
4
1 2
1 3
1 4
```

**输出**

```text
1 1 1
```

**说明**

所有叶子节点（2、3、4）到根节点距离的奇偶性相同，全部分配权限码 1 即可满足任意叶子路径异或和为 0，且字典序最小。

### 样例 2

**输入**

```text
4
1 2
2 3
3 4
```

**输出**

```text
1 2 3
```

**说明**

叶子节点为 1 和 4，路径包含三条边。为满足异或和为 0 且使用最少种类权限码，按输入顺序字典序最小的正整数分配方案为 1、2、3。

## 说明

题目保证在正整数范围内存在满足条件的解。优化目标具有严格优先级：首先最小化使用的不同权限码种类数，其次在满足该条件的所有方案中，使按通道编号顺序排列的序列字典序最小。
