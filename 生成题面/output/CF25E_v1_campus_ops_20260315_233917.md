# 综合日程码

> Variant: `CF25E` / `v1` / theme `campus_ops`

## 题目描述

校学生会正在筹备学期的三大核心活动，每个活动都有一个独特的识别码。为了方便系统统一管理，你需要生成一个综合日程码。

这个综合日程码必须是一个连续的字符串，且必须包含所有三个活动的识别码作为其子串（即每个识别码都必须连续地出现在综合日程码中）。

为了节省存储空间并提高检索效率，要求生成的综合日程码长度尽可能短。如果存在多个长度最短的方案，请选择字典序最小的那个作为最终结果。

例如，若三个识别码分别为 "abc", "bcd", "cde"，则 "abcde" 是一个合法的綜合日程碼，因为它包含了所有三个识别码，且长度为 5。

## 输入格式

输入共三行，每行包含一个非空字符串，表示一个活动的识别码。
字符串仅由小写英文字母组成。

## 输出格式

输出一行，包含一个字符串，表示满足条件的综合日程码。

## 数据范围与限制

- 每个识别码的长度不超过 100,000。
- 时间限制：1.0 秒。
- 空间限制：256 MB。
- 字符串仅包含小写英文字母 'a'-'z'。

## 样例

### 样例 1

**输入**

```text
abc
def
ghi
```

**输出**

```text
abcdefghi
```

**说明**

三个字符串没有重叠部分。将它们按字典序拼接可获得最短且字典序最小的结果。

### 样例 2

**输入**

```text
abc
bcd
cde
```

**输出**

```text
abcde
```

**说明**

字符串之间存在重叠。'abc' 与 'bcd' 重叠 'bc'，'bcd' 与 'cde' 重叠 'cd'，合并后得到 'abcde'，长度为 5。

## 说明

子串定义为原字符串中连续的一段字符序列。若有多个最优解，请输出字典序最小者。
