# 校园社团嵌套链

> Variant: `CampusNestedClubs` / `v1` / theme `campus_ops`

## 题目描述

某高校共有 n 个社团，编号为 1 至 n。社团间的日常协作关系构成了一棵无向树。为了筹备跨社团联合展演，学生会需要规划一条“社团嵌套链”。

一条合法的嵌套链定义为社团编号序列 a_1, a_2, ..., a_k，需同时满足以下核心约束：
1. 协作连通性：序列中任意相邻的两个社团 a_i 与 a_{i+1} 必须在协作网络中存在直接边相连。
2. 资源嵌套偏序：序列需体现严格的资源包含关系。在树形结构中，这意味着从 a_1 出发，每一步都必须走向一个此前未访问过的相邻社团，且路径不允许折返或重复经过节点（即序列对应树上的一条简单路径）。
3. 字典序规范优先级：在所有满足上述条件且长度 k 最大的嵌套链中，必须选择按社团编号字典序最小的序列。该规范序作为硬性构造约束，直接参与路径决策，而非事后筛选。

请根据给定的协作网络，构造并输出满足条件的最长嵌套链完整序列。

## 输入格式

第一行包含一个整数 n，表示社团总数。
接下来 n-1 行，每行包含两个整数 u 和 v，表示社团 u 与社团 v 之间存在直接协作关系。
输入保证给定的图是一棵包含 n 个节点的树。

## 输出格式

第一行输出一个整数 k，表示最长嵌套链的长度。
第二行输出 k 个整数，表示字典序最小的最长嵌套链序列，相邻整数之间用单个空格分隔。

## 数据范围与限制

- 时间限制：1.0 秒
- 空间限制：256 MB
- 3 ≤ n ≤ 10^5
- 社团编号范围为 1 到 n
- 输入保证构成一棵合法的树

## 样例

### 样例 1

**输入**

```text
5
1 2
2 3
3 4
4 5
```

**输出**

```text
5
1 2 3 4 5
```

**说明**

协作网络呈线性结构。唯一的最长嵌套链包含全部 5 个社团，序列为 1 2 3 4 5。

### 样例 2

**输入**

```text
4
1 2
1 3
1 4
```

**输出**

```text
3
2 1 3
```

**说明**

社团 1 为中心枢纽，连接 2、3、4。最长链长度为 3，必须经过中心节点连接两个叶子节点。所有长度为 3 的合法序列中，字典序最小的是 2 1 3。

## 说明

字典序比较规则为从左至右逐位对比社团编号，首次出现较小编号的序列视为更优。题目保证最长嵌套链的字典序最小解唯一。输出序列中的社团编号必须互不相同。
