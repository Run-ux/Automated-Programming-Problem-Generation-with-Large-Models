# 校园社团密钥同步计划

> Variant: `CampusOps_CanonicalXOR` / `v1` / theme `campus_ops`

## 题目描述

校园后勤部正在为各社团的活动室网络配置智能门禁系统。该网络由 $N$ 个活动室和 $N-1$ 条双向走廊组成，整体呈树状结构。为了简化物资流转的核验流程，系统要求为每条走廊分配一个正整数“通行密钥”。
出于安全协议与存储成本的考量，分配方案必须同时满足以下三项硬性指标：
1. **路径异或闭合**：对于网络中任意两个不同的“末端活动室”（即仅与一条走廊直接相连的活动室），它们之间唯一通路上的所有走廊密钥按位异或（XOR）的结果必须严格等于 $0$。
2. **极值密钥种类**：在满足路径异或闭合的前提下，所使用的不同密钥种类数量必须达到理论最小值。
3. **字典序规范输出**：在所有满足前两项指标的方案中，后勤部要求按照走廊的录入顺序（即输入数据中第 $1$ 条、第 $2$ 条……第 $N-1$ 条）提取出的密钥序列，其字典序必须最小。
请你编写程序，为后勤部生成这套符合规范的密钥分配序列。

## 输入格式

第一行包含一个正整数 $N$，表示活动室的总数。
接下来 $N-1$ 行，每行包含两个正整数 $u, v$，表示一条走廊连接的两个活动室编号。走廊将严格按照输入顺序被依次编号为 $1$ 至 $N-1$。

## 输出格式

输出一行，包含 $N-1$ 个正整数，表示按输入顺序分配的各条走廊密钥，相邻整数之间用空格分隔。

## 数据范围与限制

- $3 \le N \le 10^5$
- 保证输入数据构成一棵合法的树，且至少存在两个末端活动室。
- 分配的密钥必须为正整数，且不超过 $10^9$。
- 时间限制：1.0 秒，空间限制：256 MB。

## 样例

### 样例 1

**输入**

```text
4
1 2
1 3
1 4
```

**输出**

```text
1 1 1
```

**说明**

所有末端活动室（2, 3, 4）相对于任意根节点的深度奇偶性一致，理论最小不同密钥数为 1。为满足字典序最小，全部走廊均分配密钥 1，此时任意两末端路径异或和均为 0。

### 样例 2

**输入**

```text
5
1 2
2 3
2 4
4 5
```

**输出**

```text
1 1 2 3
```

**说明**

末端活动室（1, 3, 5）的深度奇偶性存在差异，理论最小不同密钥数为 3。按输入顺序贪心分配：前两条边（连接末端 1 与 3）优先赋 1 以合并同类需求；第三条边赋 2；第四条边根据异或闭合条件必须为 3。序列 [1, 1, 2, 3] 为满足条件的字典序最小解。

## 说明

题目保证在给定约束下合法解一定存在。字典序比较规则为：从左至右依次比较序列元素，首个不同位置数值较小的序列字典序更小。构造过程中需严格保证前缀选择不会导致剩余子树无法在最小密钥种类限制下完成异或闭合。
