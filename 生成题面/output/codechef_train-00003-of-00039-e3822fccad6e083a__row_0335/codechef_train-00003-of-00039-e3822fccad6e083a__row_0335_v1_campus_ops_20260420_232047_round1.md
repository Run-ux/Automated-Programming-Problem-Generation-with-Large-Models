# 校园社团节布线规划

> Variant: `campus_ops_canonical_wire_placement` / `v1` / theme `campus_ops`

## 题目描述

校园社团节即将开幕，学生会需要在一条直线上编号为 1 到 N 的 N 个预留点位依次搭建社团摊位。为了保障供电与网络，每次搭建新摊位时，必须拉设一根线缆连接到距离该点位最近的、已经搭建好的摊位或位于 0 和 N+1 处的两个固定信号塔。线缆长度即为本次搭建的消耗。所有摊位搭建完毕后，总消耗为各次线缆长度之和。

由于预算有限，总消耗不能超过给定的上限 M。同时，为了便于宣传与调度，学生会希望摊位的搭建顺序（即一个 1 到 N 的排列）在字典序上尽可能靠前。请找出满足预算限制的字典序最小的搭建顺序。若无论如何安排都会超出预算，请输出 -1。

## 输入格式

第一行包含一个正整数 T，表示测试用例的数量。
接下来 T 行，每行包含两个正整数 N 和 M，分别表示预留点位的数量和预算上限。

## 输出格式

对于每个测试用例，输出一行。若存在满足条件的搭建顺序，输出 N 个用空格隔开的正整数，表示该顺序；否则输出 -1。

## 数据范围与限制

- 1 ≤ T ≤ 10
- 1 ≤ N ≤ 1000
- 1 ≤ M ≤ 10^9
- 时间限制：1.0 秒
- 空间限制：256 MB

## 样例

### 样例 1

**输入**

```text
2
3 3
3 2
```

**输出**

```text
1 2 3
-1
```

**说明**

对于 N=3，最小总消耗为 3。当 M=3 时，顺序 1 2 3 的总消耗为 3，满足预算且字典序最小。当 M=2 时，任何顺序的消耗均大于 2，故无解。

### 样例 2

**输入**

```text
1
4 4
```

**输出**

```text
1 2 3 4
```

**说明**

当 N=4 时，最小总消耗为 4。预算 M=4 恰好满足最小消耗，顺序 1 2 3 4 的总消耗为 4，符合要求。

## 说明

距离定义为两点坐标差的绝对值。若存在多个距离相同的最近目标，任选其一连接，线缆长度相同。输出顺序需严格为 1 到 N 的排列。若存在多个合法顺序，必须输出字典序最小的一个。
