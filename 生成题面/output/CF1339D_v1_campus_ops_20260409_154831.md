# 社团活动室的智能门禁

> Variant: `CampusOps_CanonicalXOR` / `v1` / theme `campus_ops`

## 题目描述

学校社团联合会最近对旧教学楼进行了翻新，将走廊改造成了智能门禁通道。教学楼的结构可以看作一棵以 1 号节点为根的树，其中叶子节点代表各社团的活动室，非叶子节点代表走廊交汇点或楼梯间。为了统一管理，后勤部需要为每条走廊分配一个正整数门禁码。

出于安全与通行效率的考虑，门禁码需满足以下核心规则：任意两个不同社团活动室（叶子节点）之间的唯一路径上，所有走廊门禁码的按位异或（XOR）和必须恰好为 0。

为了降低维护成本，后勤部希望使用的不同门禁码种类数尽可能少。在满足该条件的所有合法分配方案中，还需要按照特定的巡查顺序输出门禁码序列，并要求该序列的字典序严格最小。

巡查顺序定义如下：从根节点（1 号节点）出发进行深度优先搜索（DFS），每次优先访问编号较小的相邻节点（父节点除外）。按遍历到每条边的先后顺序，依次记录该边的门禁码，形成一个长度为 $N-1$ 的序列。你的任务是求出这个字典序最小的门禁码序列。

## 输入格式

第一行包含一个整数 $N$，表示教学楼的节点总数。
接下来 $N-1$ 行，每行包含两个整数 $u, v$，表示节点 $u$ 和节点 $v$ 之间有一条走廊相连。
输入保证构成一棵以 1 为根的树，且 $N \ge 3$。

## 输出格式

输出一行，包含 $N-1$ 个正整数，表示按 DFS 巡查顺序得到的门禁码序列。相邻整数之间用单个空格分隔。

## 数据范围与限制

- $3 \le N \le 10^5$
- 门禁码必须为 $[1, 10^6]$ 范围内的整数
- 时间限制：2.0 秒
- 空间限制：512 MB

## 样例

### 样例 1

**输入**

```text
4
1 2
1 3
1 4
```

**输出**

```text
1 1 1
```

**说明**

叶子节点为 2, 3, 4，它们到根节点的深度均为 1（奇偶性相同）。此时只需将所有边权赋为 1，任意两叶子路径异或和均为 $1 \oplus 1 = 0$。不同门禁码种类数为 1，已达理论最小。DFS 访问顺序为边 (1,2), (1,3), (1,4)，对应序列为 1 1 1，字典序最小。

### 样例 2

**输入**

```text
4
1 2
1 3
2 4
```

**输出**

```text
1 2 3
```

**说明**

叶子节点为 3 和 4，深度分别为 1 和 2（奇偶性不同）。理论最少需 3 种门禁码。DFS 访问边的顺序为 (1,2), (1,3), (2,4)。贪心尝试分配：令 (1,2)=1，为满足叶子 3 与 4 路径异或为 0，需 $w(1,3) \oplus 1 \oplus w(2,4) = 0$。尝试 $w(1,3)=2$，则 $w(2,4)=3$，使用种类 {1,2,3} 共 3 种，序列为 1 2 3。该方案满足所有条件且字典序最小。

## 说明

1. 题目保证对于给定的树结构，一定存在满足条件的门禁码分配方案。
2. “不同门禁码种类数最少”是指在该树结构下，满足异或约束所需的全局最小种类数（通常为 1 或 3）。你的方案必须恰好使用该最小种类数，不可随意增加。
3. DFS 遍历时，若当前节点有多个未访问的子节点，必须严格按照节点编号从小到大的顺序进行递归。
4. 输出序列的长度严格等于 $N-1$，且每个数必须在 $[1, 10^6]$ 范围内。
