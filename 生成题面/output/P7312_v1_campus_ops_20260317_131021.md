# 社团展位分配

> Variant: `P7312` / `v1` / theme `campus_ops`

## 题目描述

大学一年一度的社团文化节即将开始，学生会需要为各个社团分配活动展位。校园地图可以看作一个巨大的平面直角坐标系，每个展位都是一个边与坐标轴平行的矩形。

共有 N 个社团按顺序提交了展位申请。第 i 个社团申请的展位左下角坐标为 (X_i, Y_i)，宽度为 A_i，高度为 B_i。由于场地有限，后提交的申请可能会覆盖先提交的申请。具体规则如下：

如果一个社团的展位区域与任何后续社团（即申请顺序在它之后的社团）的展位区域有重叠（重叠面积大于 0），则该社团的展位被视为“被遮挡”，无法正常使用。只有当一个社团的展位没有被任何后续社团遮挡时，该社团才算申请成功。

注意，展位边缘重合不算重叠，只有内部区域相交才视为遮挡。学生会希望知道，最终有多少个社团能够成功获得可用的展位。

请编写程序，根据给定的申请顺序和展位信息，计算成功获得展位的社团数量。

## 输入格式

第一行包含一个正整数 N，表示社团申请的数量。
接下来 N 行，每行包含四个整数 X_i, Y_i, A_i, B_i，分别表示第 i 个社团展位的左下角横坐标、纵坐标、宽度和高度。
输入数据保证所有坐标均为非负整数。

## 输出格式

输出一行，包含一个整数，表示成功获得展位的社团数量。

## 数据范围与限制

- 对于 100% 的数据，1 ≤ N ≤ 100,000。
- 0 ≤ X_i, Y_i ≤ 10^9。
- 1 ≤ A_i, B_i ≤ 10^9。
- 时间限制：1.0 秒。
- 空间限制：256 MB。

## 样例

### 样例 1

**输入**

```text
3
0 0 2 2
1 1 2 2
3 3 1 1
```

**输出**

```text
2
```

**说明**

第 1 个社团展位为 [0,2]×[0,2]，第 2 个社团展位为 [1,3]×[1,3]。两者在 [1,2]×[1,2] 区域重叠，因此第 1 个社团被第 2 个社团遮挡。第 2 个社团与第 3 个社团（[3,4]×[3,4]）仅在角点接触，无面积重叠，故第 2 个社团未被遮挡。第 3 个社团之后无社团，未被遮挡。成功社团为第 2、3 个，共 2 个。

### 样例 2

**输入**

```text
2
0 0 5 5
0 0 5 5
```

**输出**

```text
1
```

**说明**

第 1 个社团展位完全被第 2 个社团展位覆盖，因此第 1 个社团失败。第 2 个社团未被后续遮挡，成功。总计 1 个。

## 说明

1. 展位区域定义为左闭右开区间或连续区域，只要两个矩形内部交集面积大于 0 即视为重叠。仅边界接触不计入重叠。
2. 申请顺序即为输入顺序，编号从 1 到 N。
3. 结果不需要取模，直接输出整数即可。
