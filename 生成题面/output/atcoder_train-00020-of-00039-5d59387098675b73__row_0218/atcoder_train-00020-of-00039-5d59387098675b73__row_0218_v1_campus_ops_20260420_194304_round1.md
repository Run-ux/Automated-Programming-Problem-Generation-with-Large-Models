# 社团层级排班

> Variant: `campus_ops_tree_labeling_count` / `v1` / theme `campus_ops`

## 题目描述

某高校学生会采用严格的树形组织架构进行日常运营。组织架构以会长为根节点，其余成员按上下级关系构成一棵有根树。新学期伊始，后勤部需要为 N 名成员分配 N 个值班优先级等级（给定一个包含 N 个正整数的序列，序列中可能存在重复数字）。分配过程需严格遵守以下运营规范：
1. 精确匹配：每个优先级等级必须恰好分配给一名成员，不允许遗漏或重复使用。
2. 层级约束：在任何上下级关系中，上级的值班优先级数值必须不大于其直接下级（即父节点分配的值 ≤ 子节点分配的值）。
3. 核心锁定：会长（根节点）必须分配所有等级中的最小值，以确保全局调度基准的统一。

后勤部希望评估排班方案的灵活性。请计算满足上述所有规范的不同排班方案总数。若两种方案中至少有一名成员被分配了不同的优先级数值，则视为不同方案。由于方案数可能极大，请将结果对 10^9+7 取模后输出。

## 输入格式

第一行包含一个正整数 N，表示社团成员总数。
接下来 N-1 行，每行包含两个正整数 u 和 v，表示成员 u 与成员 v 之间存在直接的上下级关系（输入保证构成一棵以 1 号成员为根的树）。
最后一行包含 N 个正整数 a_1, a_2, ..., a_N，表示后勤部提供的值班优先级等级序列。

## 输出格式

输出一个整数，表示满足所有规范的排班方案总数对 10^9+7 取模的结果。

## 数据范围与限制

- 2 ≤ N ≤ 200000
- 1 ≤ a_i ≤ 100000
- 输入保证构成一棵以 1 为根的树
- 时间限制：1.0 秒
- 空间限制：256 MB

## 样例

### 样例 1

**输入**

```text
3
1 2
1 3
1 2 2
```

**输出**

```text
1
```

**说明**

根节点 1 必须分配最小值 1。剩余节点 2 和 3 只能分配两个 2。由于父节点值需不大于子节点值，且剩余值均为 2，仅存在唯一一种分配方式：1->1, 2->2, 3->2。

### 样例 2

**输入**

```text
4
1 2
1 3
1 4
1 2 2 3
```

**输出**

```text
3
```

**说明**

根节点 1 固定分配 1。剩余节点 2,3,4 需分配 {2,2,3}。由于它们均为叶子节点，只需将这三个数全排列分配给它们即可。考虑到两个 2 不可区分，实际不同的分配方案为 3 种（即 3 在哪个节点上）。

## 说明

结果需对 10^9+7 取模。方案差异仅取决于顶点到数值的映射关系，相同数值的交换不产生新方案。输入数据保证树结构连通且无环，根节点固定为 1 号顶点。
