# 社团物资审批

> Variant: `CampusOps_CountingTreeHalving` / `v1` / theme `campus_ops`

## 题目描述

校园社团联盟有一张由 N 个节点构成的物资调配网络，节点 1 为总仓库，其余节点为各社团活动室。网络呈树形结构，每条边代表一条运输通道，初始有一个物资容量 w。对于树中的任意一条边 e，将其删除后，不包含根节点 1 的连通块中的叶子节点（除根节点外度数为 1 的节点）数量记为 L_e。该边对全局调配压力的贡献定义为“当前容量 × L_e”。全局调配压力等于所有边的贡献之和。

为了控制压力，联盟允许对每条边独立执行若干次“分级审批”操作。每次操作会将该边的容量更新为 floor(当前容量 / 2)。操作顺序不影响最终容量。规定每条边的操作次数 c_e 为非负整数，且当容量降至 0 后继续操作不再改变容量，因此我们只考虑 0 ≤ c_e ≤ 30 的有效操作次数。

现给定一个压力阈值 S，请计算有多少种不同的操作次数分配方案 (c_1, c_2, ..., c_{N-1})，使得执行操作后的全局调配压力不超过 S。注意，只要任意一条边的操作次数不同，即视为不同的方案。由于方案数可能很大，请输出对 10^9+7 取模的结果。

## 输入格式

第一行包含一个正整数 T，表示测试用例的数量。
每个测试用例的第一行包含两个整数 N 和 S，分别表示节点数和压力阈值。
接下来 N-1 行，每行包含三个整数 u, v, w，表示节点 u 与节点 v 之间有一条初始容量为 w 的边。
数据保证输入构成一棵以 1 为根的树。

## 输出格式

对于每个测试用例，输出一行一个整数，表示合法方案数对 10^9+7 取模的结果。

## 数据范围与限制

- 1 ≤ T ≤ 10
- 2 ≤ N ≤ 5000
- 1 ≤ S ≤ 10^14
- 1 ≤ w ≤ 10^9
- 时间限制：2.0 秒
- 空间限制：512 MB

## 样例

### 样例 1

**输入**

```text
1
2 2
1 2 4
```

**输出**

```text
3
```

**说明**

边(1,2)覆盖叶子数 L=1。操作次数 c 可取 0,1,2,3，对应贡献分别为 4,2,1,0。要求总贡献 ≤ 2，故 c=1,2,3 合法，共 3 种方案。

### 样例 2

**输入**

```text
1
3 3
1 2 3
2 3 2
```

**输出**

```text
7
```

**说明**

边(1,2)与(2,3)覆盖叶子数均为 1。各自操作次数 c1,c2 ∈ {0,1,2}。枚举所有组合，仅 (0,0) 和 (0,1) 的总贡献（5 和 4）超过 3，其余 7 种组合均满足条件。

## 说明

本题要求精确统计方案数并对 10^9+7 取模。操作次数向量是唯一的计数单元，不同向量严格对应不同方案。当边权降为 0 时，继续增加操作次数不会产生新的有效状态，故每条边的有效操作次数上限为 30。全局压力计算严格遵循线性可加性，各边决策相互独立。
