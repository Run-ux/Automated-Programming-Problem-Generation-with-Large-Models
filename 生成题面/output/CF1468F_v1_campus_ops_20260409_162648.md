# 社团互视配对计划

> Variant: `CampusOps_CanonicalGazeMatching` / `v1` / theme `campus_ops`

## 题目描述

秋季社团招新日，同学们分散站在操场各处。为了高效开展破冰活动，学生会决定组织一场“互视连线”环节。每位同学入场时都会记录自己的站位坐标 (x, y) 以及目光注视的目标点 (u, v)，由此确定一个初始视线方向向量 d = (u-x, v-y)。
连线规则如下：
1. 方向约束：仅当两名同学的初始视线方向向量恰好相反（即共线且方向严格相反）时，他们才具备建立互视连线的资格。
2. 唯一性约束：每名同学在整个活动中最多只能参与一次连线。
3. 规模与规范约束：学生会希望连线的总对数尽可能多。在所有能达到最大对数的配对方案中，必须选出字典序最小的一个。字典序的比较方式为：将方案中的所有配对 (a, b) 统一调整为 a < b，并按 a 升序排列；若 a 相同则按 b 升序排列。依次比较两个方案序列中的每一对，首次出现差异的位置数值较小的方案字典序更小。
请根据给定的同学信息，计算出符合上述所有要求的连线方案。

## 输入格式

输入包含多组测试数据。第一行一个整数 T 表示测试用例数量。
对于每组数据：
第一行一个整数 N，表示参与活动的同学总数。
接下来 N 行，每行包含五个整数 id, x, y, u, v，分别表示同学的编号、站位坐标和视线目标点坐标。保证所有 id 互不相同。

## 输出格式

对于每组测试数据，输出两行：
第一行一个整数 K，表示最大可能的连线对数。
接下来 K 行，每行输出两个整数 id1 id2（保证 id1 < id2），表示一对连线。所有配对必须严格按照题目定义的字典序升序输出。

## 数据范围与限制

- 1 <= T <= 10
- 1 <= N <= 100000，所有测试用例的 N 之和不超过 200000
- 1 <= id <= 1000000000，且互不相同
- -1000000000 <= x, y, u, v <= 1000000000，且保证 (x, y) != (u, v)
- 时间限制：2.0 秒
- 空间限制：512 MB

## 样例

### 样例 1

**输入**

```text
1
4
1 0 0 1 0
2 0 0 -1 0
3 0 0 0 1
4 0 0 0 -1
```

**输出**

```text
2
1 2
3 4
```

**说明**

向量分别为 (1,0), (-1,0), (0,1), (0,-1)。1与2方向相反，3与4方向相反。最大匹配数为2。按ID升序配对后字典序最小，输出 (1,2) 和 (3,4)。

### 样例 2

**输入**

```text
1
5
10 0 0 2 2
20 0 0 -1 -1
30 0 0 3 3
40 0 0 -2 -2
50 0 0 1 0
```

**输出**

```text
2
10 20
30 40
```

**说明**

归一化后，方向(1,1)组有ID{10,30}，方向(-1,-1)组有ID{20,40}，方向(1,0)组有ID{50}。仅前两组互为相反数。为达到最大匹配数2，需将两组按ID升序依次配对：(10,20)与(30,40)。该方案在所有最大匹配中字典序最小。

## 说明

向量归一化时需注意符号统一与最大公约数约分。配对输出时务必保证每行 id1 < id2，且整体序列按字典序严格升序排列。
