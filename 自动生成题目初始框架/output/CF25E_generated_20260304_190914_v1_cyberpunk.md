### Title: 黑客入侵：最小化数据流

### Description:
在未来的赛博朋克世界里，你是一名顶尖的黑客。你的任务是渗透进一个高度加密的数据网络中，以获取关键信息。这个网络由一系列节点组成，每个节点都有其独特的加密序列。为了成功地穿透整个网络而不被发现，你需要构造一个最短的数据流，该数据流能够包含所有指定节点的加密序列，从而使得你能够一次性通过所有防火墙。

给定 $N$ 个节点的加密序列（长度为 $K$ 的字符串），你的目标是找到一个最短的数据流，使得这个数据流包含所有的加密序列。每个节点的加密序列必须至少出现一次，并且顺序不限。请记住，构建这样的数据流不仅需要考虑如何将各个序列合并，还要确保最终的数据流尽可能短，以减少被检测的风险。

### Input Format:
- 第一行包含两个整数 $N$ 和 $K$ ($1 \leq N \leq 6, 1 \leq K \leq 100$)，分别表示节点的数量和每个加密序列的长度。
- 接下来的 $N$ 行每行包含一个长度为 $K$ 的字符串，代表每个节点的加密序列。

### Output Format:
- 输出一个整数，表示能够包含所有给定加密序列的最短数据流的长度。

### Example:
- **Input**:
  ```
  3 3
  ABC
  BCA
  AAA
  ```
- **Output**:
  ```
  5
  ```

### Explanation:
在这个例子中，一种可能的最短数据流是 "AAABC" 或 "BCAAA"，它们都包含了所有的加密序列，并且长度为 5。

### Constraints:
- 时间限制：1.0s
- 内存限制：256MB
- $1 \leq N \leq 6$
- $1 \leq K \leq 100$

---

这道题目将核心逻辑中的最优子结构和贪心选择性质隐喻成黑客寻找最短路径来破解系统的过程，同时保持了算法竞赛题目的挑战性和趣味性。
