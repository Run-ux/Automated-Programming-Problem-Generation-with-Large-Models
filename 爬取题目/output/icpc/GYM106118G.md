G. Graph Game

The Forgotten Caverns form a network of $n$ chambers, connected by $m$ bidirectional tunnels. This network is a connected, undirected, unweighted graph with vertices numbered from $1$ to $n$.

Each chamber $i$ has a difficulty value $a_i$. Some chambers also contain ancient treasures, indicated by $b_i \in \{0,1\}$ : if $b_i=1$, there is a treasure in chamber $i$; if $b_i=0$, there is none.

The treasure configuration is persistent and updates after each type-2 query.

Input
The first line contains three integers $n$, $m$, $q$ ($2 \le n \le 10^5$, $n-1 \le m \le 2\cdot 10^5$, $1 \le q \le 10^5$) — the number of chambers, tunnels, and queries respectively. The second line contains $n$ integers $a_1, a_2, \dots, a_n$ ($1 \le a_i \le 10^9$) — the difficulty of each chamber. The third line contains $n$ integers $b_1, b_2, \dots, b_n$ ($b_i \in \{0,1\}$) — the initial treasure configuration. The next $m$ lines each contain two integers $x, y$ ($1 \le x, y \le n$, $x \ne y$) — a tunnel between chambers $x$ and $y$. The next $q$ lines each describe one query: For a query of type-1, the line has the form 1 u p, where $1 \le u \le n$ and $a_u \le p \le 10^9$. For a query of type-2, the line has the form 2 l r, where $1 \le l \le r \le n$. It is guaranteed that the graph is connected, with no multiple edges and no self-loops. It is guaranteed that there will be at least one type-1 query.

Output
For each query of type-1, output a single integer — the minimum number of coins required.

Constraints
time limit per test 1 second
memory limit per test 256 megabytes
