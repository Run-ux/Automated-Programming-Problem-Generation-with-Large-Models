H. Parallel Checking

The competition has ended — and now Artem only needs to check the submissions of the teams for plagiarism.

Artem has a total of $n$ submissions, and checking the $i$ -th submission will take $A_i$ quanta of time.

The problem is that the checking cannot take too long — it must be completed in no more than $T$ quanta of time.

To speed up the process, Artem can parallelize it by allocating $d$ threads. In this case, the $i$ -th submission will be checked in $\lceil\frac{A_i}{d}\rceil$ quanta of time (the result of integer division, rounded up).

Help Artem find the minimum number of threads $d$ such that $\sum{\lceil\frac{A_i}{d}\rceil} \le T$.

Input
The first line contains two integers $n$ and $T$ $(1 \le n \le 2 \cdot 10^5; n \le T \le 10^{12})$ — the number of available submissions. The second line contains $n$ integers $A_i$ $(1 \le A_i \le 10^{12})$ — the number of quanta of time required to check the $i$ -th submission.

Output
In a single line, output an integer $d$ $(1 \le d \le 10^{12})$ — the minimum number of threads such that $\sum{\lceil\frac{A_i}{d}\rceil} \le T$.

Constraints
time limit per test 2 seconds
memory limit per test 256 megabytes
