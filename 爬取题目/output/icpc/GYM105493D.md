D. Conspiracy Theory

A set of events is given by an array of integers a i.

You believe that from event number i, event number j could be anticipated if both of the following conditions are true:

A chain of events, where each subsequent event could be anticipated from the previous one, is called a conspiracy.

Prove to everyone that your conspiracy theory is not just a theory — find a conspiracy consisting of the largest number of events.

Input
The first line contains an integer n (1 ≤ n ≤ 10 6) — the number of events. The second line contains n integers a 1, a 2, ..., a n (1 ≤ a j ≤ 2·10 6) — the parameters of the events.

Output
In the first line, output an integer k (1 ≤ k ≤ n) — the maximum possible size of the conspiracy. In the second line, output k integers p j (1 ≤ p j ≤ n) separated by spaces  — the indices of these events in the order of anticipation. In case there are multiple conspiracies of maximum size, output any of them.

Constraints
time limit per test 4 seconds
memory limit per test 512 megabytes
